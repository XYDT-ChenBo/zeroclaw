# AI安全检测服务（Guardrail）完整集成方案

## 概述

本方案基于《AI安全检测服务接口文档》，在 ZeroClaw 中集成双向安全围栏功能：
- **用户输入检测**：LLM 调用前，检测用户发送的内容
- **模型输出检测**：LLM 返回后、发送给用户前，检测模型生成的内容

**检测接口**：`POST /api/aigc/security/check/text`

## 目标

- 在 LLM 调用前拦截用户输入，进行安全检测
- 在 LLM 返回后、工具解析前拦截模型原始输出，进行安全检测
- 任一检测失败时拒绝继续处理，并向用户返回明确的拒绝信息
- 默认采用 fail-closed 策略（接口异常时拒绝）
- 两个检测维度独立配置开关

## 范围

### In Scope
- 用户输入安全检测（Hook: `before_llm_call`）
- 模型输出安全检测（Hook: `before_llm_output`）
- 通过 Hook 机制统一拦截
- 配置 schema 扩展
- Gateway/Channels 错误响应透出

### Out of Scope（当前版本）
- 图片安全检测——需求明确暂不支持多媒体
- 多轮对话历史中的每条消息独立检测——仅检测最终合成的 prompt（用户输入）或完整响应（模型输出）
- 流式响应的逐块检测——等待完整响应后统一检测

## 接口规范

### 请求体通用字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tenantId` | integer | 是 | 租户 ID，从配置读取 |
| `robotCode` | string | 是 | 应用代码，从配置读取 |
| `robotType` | string | 是 | 应用类型，从配置读取 |
| `content` | string | 是 | 待检测文本（UTF-8），不截断 |
| `contentType` | string | 是 | `"userQuery"` 或 `"modelResponse"` |

### 响应体

| 字段路径 | 类型 | 说明 |
|----------|------|------|
| `code` | string | 业务码，"10000"表示成功 |
| `message` | string | 响应消息 |
| `data.securityCheckStatus` | boolean | `true`=通过，`false`=拒绝 |
| `data.commonAnomalyTags` | array | 异常标签列表，每个标签包含三级分类 |

### 拒绝原因生成

遍历 `commonAnomalyTags`，拼接为：
```
内容包含敏感信息：<一级>-<二级>-<三级>、...，请修改后重试
```
若标签为空，则用默认消息："内容存在安全风险，无法继续"。

## 架构设计

### 双阶段检测数据流

```mermaid
flowchart TD
    A[Agent 准备 messages] --> B[GuardrailHook.before_llm_call]
    B --> C[组装 prompt 文本<br/>contentType="userQuery"]
    C --> D[POST 用户输入检测]
    D -->|通过| E[调用 LLM Provider]
    D -->|拒绝| F[Cancel: 组装拒绝原因<br/>结束本轮]
    E --> G[LLM 返回原始响应]
    G --> H[GuardrailHook.before_llm_output]
    H --> I[提取 response.text<br/>contentType="modelResponse"]
    I --> J[POST 模型输出检测]
    J -->|通过| K[继续工具解析与执行]
    J -->|拒绝| L[Cancel: 返回拒绝原因<br/>结束本轮<br/>用户看到 reason 替代输出]
```

### Hook 设计

#### 1. before_llm_call（用户输入检测）

- **类型**：Modifying Hook
- **输入**：`messages: Vec<ChatMessage>`, `model: String`
- **输出**：`HookResult<(Vec<ChatMessage>, String)>`
- **逻辑**：
  1. 将 messages 拼接为 prompt 文本
  2. POST 到接口，`contentType="userQuery"`
  3. 若拒绝，返回 `Cancel(reason)`，**中止 LLM 调用**
  4. 若通过，返回 `Continue((messages, model))`

#### 2. before_llm_output（模型输出检测）——新增

- **类型**：Modifying Hook
- **输入**：`response_text: String`, `tool_calls: Vec<ToolCall>`
- **输出**：`HookResult<(String, Vec<ToolCall>)>`
- **逻辑**：
  1. 使用 `response_text` 作为 `content`
  2. POST 到接口，`contentType="modelResponse"`
  3. 若拒绝，返回 `Cancel(reason)`，**中止本轮后续处理**（不解析工具、不执行、不发送输出）
  4. 若通过，返回 `Continue((response_text, tool_calls))`

> **注意**：工具调用（`tool_calls`）在此阶段暂不检测（接口仅支持文本）。检测通过后，正常进行工具解析与执行。

### Hook 优先级建议

- `before_llm_call`：`100`（最早拦截，避免不必要的 LLM 调用）
- `before_llm_output`：`100`（尽快检测，避免工具执行）

```rust
use crate::providers::ToolCall;

impl HookRunner {
    // ... 现有方法 ...
    
    pub async fn run_before_llm_output(
        &self,
        mut response_text: String,
        mut tool_calls: Vec<ToolCall>,
    ) -> HookResult<(String, Vec<ToolCall>)> {
        for h in &self.handlers {
            let hook_name = h.name();
            match AssertUnwindSafe(h.before_llm_output(response_text.clone(), tool_calls.clone()))
                .catch_unwind()
                .await
            {
                Ok(HookResult::Continue((rt, tc))) => {
                    response_text = rt;
                    tool_calls = tc;
                }
                Ok(HookResult::Cancel(reason)) => {
                    info!(hook = hook_name, reason, "before_llm_output cancelled by hook");
                    return HookResult::Cancel(reason);
                }
                Err(_) => {
                    tracing::error!(hook = hook_name, "before_llm_output hook panicked; continuing with previous values");
                }
            }
        }
        HookResult::Continue((response_text, tool_calls))
    }
}
```

### 3. 新增 GuardrailHook 实现

**文件**：`src/hooks/builtin/guardrail.rs`

```rust
use async_trait::async_trait;
use crate::hooks::traits::{HookHandler, HookResult};
use crate::providers::{ChatMessage, ToolCall};
use serde::{Deserialize, Serialize};
use std::time::Duration;
use reqwest;
use tracing::{warn, error};

#[derive(Clone)]
pub struct GuardrailHook {
    config: super::GuardrailConfig,
}

impl GuardrailHook {
    pub fn new(config: super::GuardrailConfig) -> Self {
        Self { config }
    }
}

#[async_trait]
impl HookHandler for GuardrailHook {
    fn name(&self) -> &str { "guardrail" }
    fn priority(&self) -> i32 { 100 }

    // 用户输入检测
    async fn before_llm_call(
        &self,
        messages: Vec<ChatMessage>,
        model: String,
    ) -> HookResult<(Vec<ChatMessage>, String)> {
        if !self.config.enabled || !self.config.check_user_input {
            return HookResult::Continue((messages, model));
        }
        let prompt = self::assemble_prompt(&messages);
        match self::check(&self.config, &prompt, "userQuery").await {
            Ok(true) => HookResult::Continue((messages, model)),
            Ok(false) => HookResult::Cancel("用户输入包含敏感内容，请修改后重试".to_string()),
            Err(e) => {
                warn!(error = %e, "guardrail user input check failed");
                HookResult::Cancel("安全检测服务暂不可用".to_string())
            }
        }
    }

    // 模型输出检测
    async fn before_llm_output(
        &self,
        response_text: String,
        tool_calls: Vec<ToolCall>,
    ) -> HookResult<(String, Vec<ToolCall>)> {
        if !self.config.enabled || !self.config.check_model_output {
            return HookResult::Continue((response_text, tool_calls));
        }
        match self::check(&self.config, &response_text, "modelResponse").await {
            Ok(true) => HookResult::Continue((response_text, tool_calls)),
            Ok(false) => HookResult::Cancel("模型输出包含敏感内容，已被拦截".to_string()),
            Err(e) => {
                warn!(error = %e, "guardrail model output check failed");
                HookResult::Cancel("输出安全检测服务暂不可用".to_string())
            }
        }
    }
}

// 内部辅助函数
impl GuardrailHook {
    async fn check(
        config: &super::GuardrailConfig,
        content: &str,
        content_type: &str,
    ) -> Result<bool> {
        let client = reqwest::Client::new();
        let req = GuardrailRequest {
            tenant_id: config.tenant_id,
            robot_code: config.robot_code.clone(),
            robot_type: config.robot_type.clone(),
            content: content.to_string(),
            content_type: content_type.to_string(),
        };
        let resp = client
            .post(&config.url)
            .json(&req)
            .timeout(Duration::from_secs(config.timeout_secs))
            .send()
            .await?;
        if !resp.status().is_success() {
            return Ok(false); // fail-closed
        }
        let body: GuardrailResponse = resp.json().await?;
        Ok(body.data.security_check_status)
    }
}

fn assemble_prompt(messages: &[ChatMessage]) -> String {
    let mut parts = Vec::new();
    for msg in messages {
        let role = match msg.role.as_str() {
            "system" => "SYSTEM",
            "user" => "USER",
            "assistant" => "ASSISTANT",
            "tool" => "TOOL",
            _ => &msg.role.to_uppercase(),
        };
        parts.push(format!("{}: {}", role, msg.content));
    }
    parts.join("\n")
}

// 请求/响应类型定义
#[derive(Serialize)]
struct GuardrailRequest {
    #[serde(rename = "tenantId")]
    tenant_id: i64,
    #[serde(rename = "robotCode")]
    robot_code: String,
    #[serde(rename = "robotType")]
    robot_type: String,
    content: String,
    #[serde(rename = "contentType")]
    content_type: String,
}

#[derive(Deserialize)]
struct GuardrailResponse {
    code: String,
    message: String,
    #[serde(rename = "data")]
    data: GuardrailData,
}

#[derive(Deserialize)]
struct GuardrailData {
    #[serde(rename = "securityCheckStatus")]
    security_check_status: bool,
    #[serde(rename = "commonAnomalyTags")]
    common_anomaly_tags: Vec<AnomalyTag>,
}

#[derive(Deserialize)]
struct AnomalyTag {
    #[serde(rename = "firstLevelTagName")]
    first_level: String,
    #[serde(rename = "secondLevelTagName")]
    second_level: String,
    #[serde(rename = "thirdLevelTagName")]
    third_level: String,
}
```

## 拒绝信息展示策略

- **用户输入检测失败**：在 `before_llm_call` 返回 `Cancel(reason)`，本轮不调用 LLM。渠道向用户显示 reason（例如："用户输入包含敏感内容，请修改后重试"）。
- **模型输出检测失败**：在 `before_llm_output` 返回 `Cancel(reason)`，本轮不再解析工具、不执行、不发送。渠道向用户显示 reason（例如："模型输出包含敏感内容，已被拦截"），**用户看不到模型原始输出**。

## 安全与隐私

1. **数据外泄风险**：GuardrailHook 会将完整 prompt/响应 发送到外部接口，必须确保：
   - 与业务方签订数据处理协议
   - 传输使用 HTTPS
   - 接口所在网络具备访问控制（如 IP 白名单）
2. **Fail-Closed**：默认拒绝策略可能导致可用性下降，但避免绕过。生产环境建议先白名单测试。
3. **错误信息最小化**：返回用户的 `reason` 避免包含堆栈、内部路径、敏感配置等。
4. **模型输出检测风险**：输出可能包含用户之前提供的敏感信息或工具结果，开启前需评估合规性。`check_model_output` 默认 `false`。
